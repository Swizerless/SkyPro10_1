## Домашка по модулю 11.2 SkyPro
![SkyPro Logo](https://digital-academy.ru/foto/school/skypro-2.png )

IT-отдел крупного банка делает новую фичу для личного кабинета клиента. Это виджет, который показывает несколько последних успешных банковских операций клиента.

### Модуль `widget.py`
В модуле `widget.py` можно найти две функции:

1. **`mask_account_card`**
   - Функция принимает один аргумент — строку, содержащую тип и номер карты или счета,  
     а возвращает строку с замаскированным номером.

2. **`get_date`**
   - Функция принимает на вход строку с датой в странном формате  
     и возвращает строку с датой в формате (ДД.ММ.ГГГГ).

### Модуль `processing.py`
В модуле `processing.py` можно найти еще две функции:

1. **`filter_by_state`**
   - Функция принимает список словарей и опционально значение для ключа `state` (по умолчанию `'EXECUTED'`),  
     а возвращает новый список словарей, содержащий только те словари,  
     у которых ключ `state` соответствует указанному значению.

2. **`sort_by_date`**
   - Функция принимает список словарей и необязательный параметр,  
     задающий порядок сортировки (по умолчанию — убывание), а возвращает  
     новый список, отсортированный по дате (`date`).

### Модуль `generators.py`
В модуле generators.py содержится реализация генераторов для работы с транзакциями и номерами банковских карт. 
В модуле можно найти следующие функции:

1. **`filter_by_currency`**
   - Функция принимает список транзакций и код валюты (например, "USD"),
а возвращает итератор, содержащий только те транзакции, которые выполнены в указанной валюте.
   - Пример использования:
```python
usd_transactions = filter_by_currency(transactions_list, "USD")
for transaction in usd_transactions:
    print(transaction)
```

2. **`transaction_descriptions`**
   - Генератор, который принимает список транзакций и возвращает описания (description) каждой транзакции.
   - Пример использования:
```python
descriptions = transaction_descriptions(transactions_list)
for desc in descriptions:
    print(desc)
```

3. **`card_number_generator`**
   - Генератор, который создает номера банковских карт в диапазоне от start до end включительно.
Номера формируются в формате XXXX XXXX XXXX XXXX.
   - Пример использования:
```python
card_numbers = card_number_generator(1000000000000000, 1000000000000002)
for number in card_numbers:
    print(number)
```

### Модуль `decorators.py`
Модуль содержит реализацию декоратора `log`, который используется для логирования вызовов функций. 

#### **Основные возможности:**
- Логирование начала выполнения функции с указанием её имени и входных параметров.
- Логирование успешного завершения функции с результатом.
- Логирование ошибок, возникающих при выполнении функции.
- Поддержка вывода логов:
  - В консоль (если параметр `filename` не указан).
  - В файл (если параметр `filename` указан).

#### **Аннотации типов:**
Для всех функций и методов в модуле добавлены аннотации типов, что обеспечивает совместимость с инструментами статической проверки типов, такими как `mypy`.

#### **Пример использования:**
```python
from decorators import log

@log()
def add(a, b):
    return a + b

@log(filename="mylog.txt")
def divide(a, b):
    return a / b

add(3, 5)       # Логи выводятся в консоль
divide(10, 2)   # Логи записываются в файл "mylog.txt"

Для использования функций достаточно импортировать их в коде и вызвать, задав параметры (см. аннотации).
```

## Инструкция по установке и настройке проекта

### 1. Клонирование репозитория
Склонируйте репозиторий на ваш компьютер:
```bash
git clone https://github.com/Swizerless/SkyPro10_1.git
```
### 2. Установка Poetry
Следуйте [официальным инструкциям](https://python-poetry.org/docs/#installation) или любым другим инструкциям по установке Poetry

### 3. Установка зависимостей
Poetry автоматически создаст виртуальное окружение и установит все зависимости из файла pyproject.toml. 

Выполните команду:
```bash
poetry install
```

### 4. Активация виртуального окружения
Чтобы активировать виртуальное окружение, созданное Poetry, выполните:
```bash
poetry shell
```

### 5. Дополнительные инструкции
- Если вы хотите добавить новые зависимости, обновите файл pyproject.toml и выполните: 
```bash
poetry add package_name
```
- Для обновления всех зависимостей выполните:
```bash
poetry update
```

### 5. Добавлен пакет тестирования для всех модулей
- Для проверки Code Coverage выполните в терминале poetry:
```bash
pytest --cov
```
